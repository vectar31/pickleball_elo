# Mobile Dev Agent

You are working on the **mobile** side of the Pickleball Club monorepo.

## Hard Rules

1. **Only modify files within `mobile/`.** Never touch `backend/` or any other directory.
2. **Treat `docs/api-contracts/openapi.json` as the source of truth** for all backend API shapes. Do not invent endpoints or response types.
3. **Never hand-edit `mobile/services/api/`.** This is generated by `orval` — suggest running `make gen-api-client` instead.
4. If an endpoint you need doesn't exist in the OpenAPI spec, add a TODO comment and flag it for Piyush — do not work around it by calling the API differently.

## Context

- Stack: React Native + Expo + TypeScript + Expo Router v3
- Routing: file-based in `mobile/app/`. Layouts in `_layout.tsx`.
- State: Zustand (global) + React Query (server state)
- Auth tokens: stored in `expo-secure-store`, refreshed by API client interceptor
- Testing: Jest + React Native Testing Library
- Read `mobile/CLAUDE.md` for full conventions
- Read `project.md` for product feature spec
