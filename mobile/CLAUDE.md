## Routing & Auth Gate

Root `_layout.tsx` pattern:
1. Hydrate `authStore` from `expo-secure-store` on app start
2. If no token → redirect to `/(auth)/login`
3. If token + no profile → redirect to `/(auth)/onboarding`
4. Otherwise → `/(tabs)/`

Use Expo Router's `<Redirect />` inside the root layout, not nested navigators.

---

## Auth Flow

- **Google SSO:** `expo-auth-session` with `Google.useAuthRequest` + `expo-web-browser`
- **Apple Sign-In:** `expo-apple-authentication`
- On success: exchange ID token with backend (`POST /auth/google` or `POST /auth/apple`) → receive JWT
- Store JWT in `expo-secure-store` under key `auth_token`
- `lib/apiClient.ts`: Axios instance that reads token from store and sets `Authorization: Bearer <token>` on every request
- Token refresh: backend returns a refresh token; interceptor catches 401 and refreshes automatically

## API Client

`services/api/` is generated by `orval` — regenerate with:
```bash
make gen-api-client   # from repo root
```

Current live API: `https://pickleball-elo.fly.dev`
Existing endpoints (Piyush's v1 — will evolve):
- `POST /auth/login` — OAuth2 password flow
- `GET /players` — list players
- `GET /ratings/singles` / `GET /ratings/doubles` — ELO ratings
- `GET /matches/singles` / `POST /matches/singles` — singles matches
- `GET /matches/doubles` / `POST /matches/doubles` — doubles matches
- `GET /stats/singles` / `GET /stats/doubles` / `GET /stats/player/{name}` — stats

**Many endpoints needed for v1 do not exist yet** (communities, user profiles, auth via Google/Apple, match validation). Mark with `// TODO(api):` and build the UI shell — Piyush will add the endpoints.

## State Management

**Zustand** for auth + UI state only. All server data goes through **React Query**.

`authStore.ts`:
```ts
{ token: string | null, userId: string | null, setAuth, clearAuth }
```

`communityStore.ts`:
```ts
{ activeCommunityId: string | null, setActiveCommunity }
```

React Query keys convention: `['communities']`, `['community', id]`, `['matches', communityId]`, `['leaderboard', communityId]`, `['profile', userId]`

## Conventions

- **No default exports on shared components.** Named exports only.
- **Screen files** (in `app/`) can use default exports (Expo Router requires it).
- **Props interfaces** named `<ComponentName>Props`, defined above the component.
- **No inline styles** — use NativeWind classes or `StyleSheet` if NativeWind doesn't cover it.
- **Error boundaries** on each screen — catch API errors gracefully, show retry UI.
- **Loading states** — every React Query call shows a skeleton, not a spinner.
