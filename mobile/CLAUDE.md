## Project Structure

```
mobile/
├── app/                  # Expo Router — file-based routing
│   ├── (auth)/           # Auth screens (login, onboarding)
│   ├── (tabs)/           # Main tab navigator
│   │   ├── index.tsx     # Home
│   │   ├── communities.tsx
│   │   └── profile.tsx
│   └── community/[id].tsx
├── components/           # Shared UI components
├── hooks/                # Custom React hooks
├── services/
│   └── api/              # Generated typed client (orval) — DO NOT hand-edit
├── stores/               # Zustand stores (or chosen state lib)
└── .env.example
```

---

## Conventions

- **Routing:** Expo Router v3, file-based. Layouts in `_layout.tsx`.
- **Components:** functional, typed props, no default exports on shared components.
- **State:** TBD (Zustand preferred for global state, React Query for server state).
- **API client:** lives in `services/api/` — generated by `orval` from `docs/api-contracts/openapi.json`. Run `make gen-api-client` after any backend spec update. Never hand-edit.
- **Auth tokens:** stored in `expo-secure-store`. Refresh handled by API client interceptor.
- **Styling:** StyleSheet or NativeWind (TBD). Be consistent within a screen.
- **Testing:** Jest + React Native Testing Library.
